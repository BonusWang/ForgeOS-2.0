## Context / 背景

移动端当前使用 hash/history 维护四个主模块，已经支持返回到上一模块。新的轻闭环升级又增加了多个临时 UI：今日任务输入面板、推进页周复盘、收纳安排、记录 composer、系统完整工具展开。为了符合 Android 返回优先级，需要让这些临时 UI 在打开时注册历史入口，并在 `popstate` 时关闭自身。

## Goals / Non-Goals / 目标与非目标

**Goals / 目标:**

- 为移动端临时面板提供统一的返回关闭机制。
- 将主要移动临时面板接入该机制。
- 为移动主导航、任务操作、待澄清操作和同步/健康状态补强可访问名称。
- 使用结构测试锁定返回和无障碍边界。

**Non-Goals / 非目标:**

- 不重写移动路由系统。
- 不新增第五个移动模块。
- 不改变数据模型、同步载荷或持久化字段。
- 不引入新的 UI 库或手势库。
- 不实现完整端到端真机截图自动化；无 ADB 设备时继续按现有降级策略记录。

## Decisions / 设计决策

1. **使用共享 hook 处理临时面板返回。**
   - 选择：新增 `useMobilePanelHistory(isOpen, closePanel, panelId)`。
   - 理由：各面板仍保留自己的状态，不需要提升到全局 store，也不破坏四模块历史。
   - 未选择：把所有弹层状态移动到 `MobileAppShell`。该方案会扩大组件耦合。

2. **只处理移动端临时面板。**
   - 选择：打开面板时 `pushState`，收到 `popstate` 时调用关闭回调。
   - 理由：满足 Android 返回“先关弹层”的核心体验。
   - 未选择：新增完整路由。当前产品不需要每个面板成为可分享 URL。

3. **无障碍增强使用原生属性。**
   - 选择：补 `aria-label`、`aria-live`、`role="status"` 等原生属性。
   - 理由：避免引入依赖，符合已有 React/CSS 结构。

## Risks / Trade-offs / 风险与取舍

- 手动点击关闭可能留下一条临时历史记录 → 主要目标是系统返回优先关闭面板；手动关闭后仍保持页面可用，后续真机体验可再细化。
- 结构测试不能验证真实 TalkBack 发音 → 先锁定代码结构，真机复审时再用 TalkBack 验证。
- 所有面板接入 hook 可能增加闭包依赖 → hook 内使用 ref 保存关闭回调，减少重复 push。
