## Why / 背景

当前暖色参考风格已经能提供一层视觉皮肤，但名称和视觉方向需要统一为 Claude 风格，并严格参考 `docs/DESIGN/DESIGN-claude.md` 的视觉系统。上一次 demo 暴露出一个风险：如果把参考风格当成页面重设计，会误改 Forge-OS 已确认的排版、模块顺序和业务工作流。

本次变更需要把边界写清楚：Claude 风格只改变视觉表现，不改变现有页面排版、业务模块集合、DOM 业务结构、数据源或操作语义。

## What Changes / 变更内容

- 将当前可见的暖色参考风格名称调整为 `Claude`，后续 UI 文案和规格描述统一使用 Claude 作为用户可见风格名。
- Claude 风格直接替换现有暖色参考风格，不保留两套相似风格，也不新增额外相似风格。
- 内部类型值、CSS class、持久化的新写入值和测试描述应从旧命名迁移为 `claude`；读取旧存储值时需要兼容并自动映射为 `claude`，避免用户已有风格偏好失效。
- Claude 风格覆盖现有暖色参考风格覆盖过的桌面端和手机端网页呈现。
- Claude 风格严格参考 `docs/DESIGN/DESIGN-claude.md` 的视觉系统，包括暖奶油画布、珊瑚色主行动点、浅色卡片表面、深色产品表面、细边框、8/12/16 圆角层级和编辑感标题层级。
- Claude 风格使用开源或系统替代字体实现视觉气质，不引入商业字体文件；标题可采用 `Cormorant Garamond`、`EB Garamond` 或 Georgia fallback，正文使用系统 sans 或项目已有可用字体。
- Claude 风格不得照抄参考品牌的品牌内容、Logo、品牌文案或产品叙事，只借鉴视觉系统。
- Claude 风格只允许调整颜色 token、字体栈、圆角、边框、表面色、按钮状态、焦点态、卡片质感、空状态视觉和轻量交互反馈。
- Claude 风格不得新增、删除、复制、重排业务模块，不得改变周看板、周复盘、反思库、月度 OKR、系统页、模块管理和主题切换的入口、顺序、布局职责或业务功能。
- Demo 与后续实现必须以“同一排版，仅换视觉皮肤”的方式表达，不再展示与当前 Forge-OS 排版不一致的新 dashboard 结构。
- 不包含破坏性变更；业务数据、持久化格式、Electron IPC、同步逻辑、任务逻辑、反思逻辑和模块启用逻辑均保持不变。

## Capabilities / 能力范围

### New Capabilities / 新增能力

无。

### Modified Capabilities / 修改能力

- `系统页面视觉样式`: 将暖色参考风格的用户可见名称和要求调整为 Claude 风格，并新增“风格变更不得改变排版、业务模块集合、DOM 业务结构和业务功能”的强约束。
- `手机端网页适配`: 如果移动端存在风格名称或风格切换呈现，也需要同步使用 Claude 名称，并保持移动端既有页面结构和业务行为不变。

## Impact / 影响范围

- 影响 OpenSpec 规格：`系统页面视觉样式`，以及可能受风格命名影响的 `手机端网页适配`。
- 影响后续实现的范围应限制在风格名称、视觉 token、CSS 外观层、焦点/激活态、空状态处理和相关测试断言。
- 不影响数据模型、store 结构、持久化格式、同步引擎、Electron IPC、任务拖拽逻辑、反思保存逻辑、模块注册表或模块启用逻辑。
- 不应修改已确认的页面排版方向：左侧导航、主页面容器、周看板主画布、今日执行条、辅助沉淀区、系统页模块集合和模块管理语义必须保持稳定。
