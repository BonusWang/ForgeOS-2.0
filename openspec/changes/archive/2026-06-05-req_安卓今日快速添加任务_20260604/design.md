## Context / 背景

移动端「今日」模块已经以 Android/手机视口为主，展示今日主线、状态、进度和今日承诺。当前任务数据来自 `useAppStore.tasks`，桌面周看板和移动端共用同一套 `addTask(content, date)` 写入路径。用户在手机端看到新任务时，还缺少就地新增入口。

## Goals / Non-Goals / 目标与非目标

**Goals / 目标:**

- 在移动端「今日」模块提供单手可触达的 `+` 新增任务入口。
- 点击入口后出现底部输入面板，适配 Android 软键盘和底部导航空间。
- 保存后写入今天日期的任务列表，并立即出现在今日承诺列表中。
- 空输入不创建任务，避免误触生成无意义任务。

**Non-Goals / 非目标:**

- 不改桌面任务看板的新增任务方式。
- 不新增任务分类、能力点选择、日期选择或复杂编辑流程。
- 不修改任务数据结构、持久化格式或 COS 同步载荷。

## Decisions / 设计决策

- 使用右下角悬浮圆形 `+` 作为入口。相比把输入框常驻在「今日承诺」卡片内，悬浮入口更接近截图中的安卓移动端使用习惯，也不会挤占主线和承诺列表的扫读空间。
- 点击后使用底部弹层式输入面板。相比全屏弹窗，底部面板保留用户所处的「今日」上下文；相比浏览器 `prompt`，它能继承应用视觉、触控尺寸和可访问语义。
- 任务写入复用 `addTask(taskInput.trim(), today)`。这样新增任务和桌面周看板、今日进度、COS 同步保持同一数据流，不引入移动端专属任务模型。
- 面板只提供取消和添加。能力点、明天/收纳等操作仍留给已有任务行按钮，避免新建流程膨胀。

## Risks / Trade-offs / 风险与取舍

- 底部面板可能遮挡底部导航或被安全区影响 → 使用 `position: fixed`、`env(safe-area-inset-bottom)` 和明确的底部间距。
- Android 软键盘弹起时视口高度变化 → 输入面板宽度限制在视口内，内容短小，按钮保持 44px 以上触控高度。
- 用户误触 `+` → 面板提供取消按钮，空输入不会保存。
- 当前「最多盯住三件事」只展示前三个 active 承诺 → 新任务仍写入今日任务；如 active 超过 3 个，今日进度会计数，详细排布仍由桌面/推进页承载。
