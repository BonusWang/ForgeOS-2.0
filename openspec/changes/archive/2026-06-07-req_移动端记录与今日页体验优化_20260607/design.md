## Context / 背景

手机端「记录」页当前把灵感和反思放在同一个纵向滚动页面里，灵感区已经有递进节点，反思区却使用另一套卡片和保存状态结构。用户需要向下滚动才能进入反思，并且同一页面出现两种交互模式，导致学习成本和视觉割裂。

手机端「今日」页当前顶部状态卡较厚，今日承诺区域使用大卡套小卡，右下角大型新增按钮抢占视觉注意力。页面可用但不够像移动端行动台。

## Goals / Non-Goals / 目标与非目标

**Goals / 目标:**

- 将「记录」页改为「灵感 / 反思」页内分段切换，减少纵向寻找成本。
- 让灵感和反思共用同一套交互框架：标题、说明、收起/展开、纵向流程链、输入、最近记录。
- 明确灵感和反思只复用框架，不复用内容语义；两者文案、字段、记录列表和保存路径必须独立。
- 将灵感步骤呈现为真正的纵向流程链，而不是普通 01/02/03/04 块列表。
- 将「今日」页改为轻量行动台：顶部「今日焦点」状态卡，下方独立「今日承诺」任务区。
- 缩小新增入口视觉权重，但保留手机端快速新增今日任务能力。

**Non-Goals / 非目标:**

- 不修改底层任务、灵感、反思数据模型。
- 不改变 COS 同步载荷或跨端同步语义。
- 不重做桌面端记录页、反思库或周看板。
- 不新增复杂动效、通知、Widget 或 Android 原生页面。
- 不改变底部主导航结构。

## Decisions / 设计决策

### 1. 记录页使用页内分段切换，而不是上下堆叠

选择「灵感 / 反思」分段控件作为记录页内部切换入口。切换后下方只展示当前类型的卡片、流程链、输入区和最近记录。

理由：灵感和反思都是「记录」页中的采集行为，但认知目标不同。分段切换能保持两者同级可达，又避免用户向下滚动寻找反思。相比继续上下堆叠，分段切换更符合手机端短路径操作。

备选方案是继续上下堆叠两个模块，但它会保留当前的滚动成本和交互割裂，不采用。

### 2. 复用交互框架，不复用内容语义

灵感和反思可以共享同一个移动端 capture shell：标题区、说明区、折叠入口、流程链容器、输入区、最近记录区。但传入的步骤、文案、字段、标签和保存逻辑必须按类型分开。

灵感步骤为：`01 想法`、`02 来源`、`03 标签`、`04 确认`。反思步骤为：`01 障碍`、`02 方法`、`03 有效/无效`、`04 确认`。灵感最近记录只展示灵感，反思最近记录只展示反思。

这样可以降低实现重复，同时避免用户看到「反思库」里出现灵感文案，或灵感流程里出现反思字段。

### 3. 流程步骤使用纵向流程链

流程链需要有一条连续的纵向关系：节点编号、节点连线、当前节点高亮、已完成状态和未开始弱化状态。数字不应只是卡片左侧的装饰块。

理由：用户把当前操作理解为一步一步完成的流程，而不是四张相似卡片。来源和标签仍可跳过，避免灵感捕捉变重。

### 4. 今日页改成轻量行动台

今日页保留两个主要区块：

- 顶部「今日焦点」状态卡：日期/周几、页面定位、今日焦点、心境、能量、完成数、进度条、今日原则。
- 下方「今日承诺」任务区：标题、`最多盯住三件事` 副文案、小型 `+` 添加入口、紧凑任务行。

不再把今日承诺做成厚重卡片套任务卡，也不再使用右下角大型悬浮按钮。新增入口放在「今日承诺」标题右侧，使按钮和当前任务区建立明确关系。

### 5. 保持功能可用性优先

本次是 UI/交互优化，不允许牺牲已有保存路径。灵感保存仍调用灵感库路径，反思保存仍调用每日反思路径，今日任务新增仍进入今天日期的任务数据流。任何文案或布局变化都必须通过结构测试、浏览器手机视口检查和 Android 构建验证。

## Risks / Trade-offs / 风险与取舍

- [Risk] 分段切换隐藏另一个类型的内容，用户可能不知道反思是否已写。  
  [Mitigation] 在分段控件或当前面板状态中展示轻量状态，例如「反思 · 已写」或最近反思数量。

- [Risk] 复用 capture shell 后文案或字段误串，造成灵感/反思混淆。  
  [Mitigation] 用类型化配置和结构测试校验每个 tab 的步骤标题、输入提示、保存目标和最近记录标题。

- [Risk] 将新增按钮从大型 FAB 改为小按钮后可发现性下降。  
  [Mitigation] 小按钮必须位于「今日承诺」标题右侧，并保留足够触控热区；任务为空时可在空态中提供更明显的添加引导。

- [Risk] 视觉减重时触控区域变小。  
  [Mitigation] 视觉尺寸可以收敛，但交互热区必须保持手机可点，按钮和任务行不得小于常见移动端可触达尺寸。
