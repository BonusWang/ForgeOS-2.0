# First Version Page Optimization Design / 第一版本页面优化设计

## Context

> 中文：背景

第一版本页面优化的目标是让产品从“ASCII 风格实验页面”转向更明确的个人成长产品：`Forge`。品牌语义从被动记录变成主动锻造，因此页面需要同时完成三件事：

- 品牌名称和 slogan 统一。
- 中文用户能直接理解每个功能模块。
- 页面卡片排列更规整，减少松散、错位和标题风格不一致。

## Decisions

> 中文：设计决策

1. 顶部品牌区显示 `Forge`，并在新风格下展示 `Forge yourself —— 自己锻造自己`。
2. 保留现有导航入口和交互行为，不新增独立页面或复制业务逻辑。
3. 模块标题使用中文短标题，避免继续暴露英文标题。
4. 页面布局优先使用统一的最大宽度、统一间距和网格/双栏结构，让主要模块边界对齐。
5. 保持 orbit 风格下的暖色背景、浅色卡片和细边框；原风格只做必要品牌和中文标题同步，不改变功能。
6. 品牌区使用浏览器可识别的非翻译标记，避免 `Forge` 被误译为普通英文动词。
7. orbit 页头只保留短标签、页面标题和状态卡片，不展示解释实现方式的 summary 文案。

## Risks

> 中文：风险

- 部分英文标题可能散落在功能组件内部，需要用搜索覆盖主要页面。
- 如果只改标题不改布局，视觉上仍会显得“换字不换结构”，所以需要同时检查卡片宽度、间距和响应式规则。
- 英文品牌词容易被浏览器翻译工具误判，需要显式标注不可翻译。
- 页头说明句会让产品界面显得像开发注释，需要从展示层移除。
- OpenSpec 的 `Requirement`、`Scenario`、`MUST` 等关键字必须保留英文，否则会影响校验。
