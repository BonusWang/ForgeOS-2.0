## Context / 背景

项目已完成周看板版本 B 排版：今日执行条、周看板区、辅助沉淀区。用户现在希望继续优化主界面范围内的页面排版，包括顶部导航、周看板、反思库、周复盘、系统页以及风格/模块/主题按钮。

Taste Skill 在本次作为设计审计参考：先统一信息层级和布局规则，避免装饰性新增内容，保持生产力工作台的信息密度和业务 DOM 稳定。

## Goals / Non-Goals / 目标与非目标

**Goals / 目标:**

- 顶部导航按钮尺寸、间距、激活态、focus 态更稳定。
- 四个主页面共享统一页面外壳、最大宽度和间距节奏。
- 反思库、周复盘、系统页的分栏布局在桌面端更整齐，窄屏自然降级。
- classic 和 orbit 只表现为视觉差异，不改变页面结构。
- 保持所有业务组件和数据逻辑原样复用。

**Non-Goals / 非目标:**

- 不新增营销式 hero、解释性 KPI、装饰性大卡片或新的业务模块。
- 不修改 store、持久化、Electron IPC、数据迁移或模块内部业务处理。
- 不重写 OKR、能力、周复盘、系统备份、更新检查、模块选择器或主题切换逻辑。
- 不改变导航入口文案和业务页面路由状态。

## Decisions / 设计决策

- 使用共享 CSS 类承载页面外壳和布局节奏。
  - 理由：主页面都处在同一个应用工作台中，统一容器比每页内联样式更容易保持一致。
  - 备选方案：继续维护各页面独立 inline layout；不采用，因为容易产生宽度和间距漂移。

- 页面层只重排容器，不进入业务组件内部。
  - 理由：本次需求是排版和交互呈现优化，业务组件内部已经承载具体操作。
  - 备选方案：逐个重构业务组件；不采用，因为改动面过大且容易触碰业务逻辑。

- 导航保留原入口文案，但通过 class 统一按钮呈现。
  - 理由：用户明确列出了导航范围，优化应提高可扫性而不是改变功能含义。
  - 备选方案：改成图标导航或侧边栏；不采用，因为会改变信息架构和使用习惯。

- 反思库、周复盘、系统页保持各自模块集合，只调整分栏宽度和间距。
  - 理由：这些页面已有清晰业务模块，主要问题是排版一致性，不是业务能力缺失。

- 周看板顶部今日模块使用紧凑状态栏，而不是四等分大卡片。
  - 理由：今日进度、每日反思、时间块和情绪追踪是辅助状态，周任务看板才是桌面端主工作区；压缩顶部高度可以减少空白并让任务列更早进入首屏。
  - 备选方案：将情绪追踪放入右侧栏；不采用，因为会压缩周看板横向空间，和用户反馈的“看板收得很窄”冲突。

- Supabase 参考风格作为第三套纯视觉模式接入。
  - 理由：用户提供的 `Supabase-showcase` 参考强调深色开发者产品、绿色强调色、圆角表面和系统字体，适合作为 Forge-OS 的现代开发者工作台风格。
  - 实现边界：只扩展 `VisualStyle`、视觉 token、shell class、按钮提示和 CSS 表现，不复制业务组件、不新增页面状态、不改变 store 或持久化。
  - 验证要求：classic、Orbit、Supabase 三种风格下，周看板业务模块数量和页面级横向溢出必须保持稳定。

- 左侧主导航按使用频率和工作流阶段排序为周看板、周复盘、反思库、系统。
  - 理由：用户的核心路径是先执行、再复盘、再沉淀检索，系统维护入口保持最后。
  - 备选方案：保持反思库在周复盘之前；不采用，因为周复盘是周执行之后的直接闭环动作。

- 周看板顶部不再放置周复盘快捷按钮。
  - 理由：左侧导航已经提供稳定周复盘入口，工具栏重复按钮会增加视觉噪音，并把周切换和复盘动作混在一起。
  - 实现边界：只删除工具栏入口和相关透传 props，不删除周复盘页面或周复盘数据逻辑。

- 模块管理弹窗改为可扫描设置面板。
  - 理由：原弹窗是窄列表加符号开关，启用状态和模块归属不够清楚；两列卡片能更快判断哪些模块在主区、哪些在侧栏。
  - 实现边界：每个模块卡片仍调用原有 `toggleModule`，不改变 `MODULE_REGISTRY`、默认启用规则或持久化结构。

## Risks / Trade-offs / 风险与取舍

- 过度统一可能压掉页面差异 → 使用共享外壳，但保留每页自身分栏比例。
- 导航按钮从 inline hover 迁移到 CSS 可能改变手感 → 保留原 onClick 和状态逻辑，只统一呈现。
- 既有未提交改动较多 → 只触碰本次相关页面和样式，不回滚其他文件。
- 截图验证可能受浏览器工具波动影响 → 以 DOM 结构、宽度、溢出和构建/lint 作为主要验证。
- 第三风格可能让原复古风格被覆盖 → 将 classic token 明确保留为默认值，并用浏览器检查 `classic -> Orbit -> Supabase -> classic` 循环。
- 模块弹窗卡片化后可能显得过重 → 启用态使用左侧细线和状态文字，而不是大面积强调边框。
