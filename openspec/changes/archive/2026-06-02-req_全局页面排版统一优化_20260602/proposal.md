## Why

主界面四个业务页面和顶部导航目前各自使用不同的宽度、分栏和间距策略，电脑端扫描体验不够统一。需要参考 Taste Skill 的设计审计思路，将页面排版收敛为稳定、克制、工作台优先的布局系统，但不改变任何业务处理逻辑。

## What Changes

- 统一顶部导航的按钮尺寸、间距、激活态和横向滚动表现，使 `[周看板]`、反思库、周复盘、◇ 系统、`[◇ 风格切换]`、`[⊕ 模块]`、主题按钮在桌面端更稳定可扫。
- 为周看板、反思库、周复盘、系统页引入一致的页面外壳、最大宽度、分栏网格、模块间距和窄屏降级规则。
- 继续保留已确认的周看板版本 B 基础结构，不让风格切换改变页面形态。
- 反思库、周复盘、系统页只优化页面层排版，不修改 OKR、能力、周复盘编辑、更新、备份、真相语录等模块内部业务逻辑。
- classic 和 orbit 风格共享同一业务 DOM 结构，仅由视觉 token 和 CSS 呈现差异。
- 在不改变业务页面集合和操作逻辑的前提下，新增参考 `Supabase-showcase` 的第三套深色开发者产品风格。
- 将应用导航收敛为左侧固定菜单，并按核心使用路径调整为 `01 周看板`、`02 周复盘`、`03 反思库`、`04 系统`。
- 移除周看板顶部重复的 `[周复盘]` 快捷按钮，避免同一功能在主工作区内制造干扰入口。
- 对模块管理弹窗进行设计审计后重做为两列设置面板，增强模块状态、区域归属和启用数量的可扫描性。
- 不新增业务模块、不删除业务模块、不改变导航文案、不改 store、持久化、Electron IPC 或数据处理逻辑。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `系统页面视觉样式`: 增加全局页面排版统一要求，覆盖顶部导航、周看板、反思库、周复盘和系统页的页面层布局边界。
- `系统页面视觉样式`: 扩展风格切换为 classic、Orbit、Supabase 三种视觉模式，并要求三种模式共享同一业务页面结构。

## Impact

- 影响界面代码：`src/App.tsx`、`src/pages/Dashboard.tsx`、`src/pages/Reflection.tsx`、`src/pages/WeeklyReview.tsx`、`src/pages/System.tsx`。
- 可能影响全局样式：`src/index.css` 中的页面外壳、导航、栅格和响应式规则。
- 不影响业务逻辑、状态结构、持久化格式、API、Electron IPC、依赖包或数据迁移。
- 本次新增风格仅增加视觉 token、样式覆盖和切换提示，不改变模块启用、任务、反思、周复盘、情绪追踪或系统设置的数据流。
- 本次模块管理弹窗只调整呈现结构和可点击区域，仍使用原有 `enabledModules` 和 `toggleModule` 业务逻辑。
