## Why / 背景

Forge-OS 目前主要面向桌面 Electron 和网页开发入口，用户在手机上只能依赖网页适配，无法获得独立 Android App 的安装、离线使用和移动端系统集成体验。

随着本地数据已经成为任务、反思、情绪和周复盘的核心资产，新增腾讯云 COS 同步可以让用户在桌面端和 Android 端之间保持数据连续，并降低单机数据丢失风险。

## What Changes / 变更内容

- 新增 Android 移动端 App 交付能力，支持在 Android 设备上安装、启动和使用 Forge-OS 的核心功能。
- 新增腾讯云 COS 数据同步能力，用于在本地数据和云端对象存储之间同步 Forge-OS 数据。
- COS 同步配置支持本地保存和再次修改，用户可以自定义 endpoint、region、bucket、objectPrefix、签名服务地址，也可以手动输入 Access Key ID 和 Secret Access Key。
- 数据备份卡片继续保留本地“导出数据 / 导入数据”，并新增“恢复历史数据”入口用于查看最近 5 次云端同步节点后选择恢复；云端立即同步入口统一放在 COS 数据同步面板。
- 数据备份卡片在说明文案上方展示只读“本机存储url”，让桌面端和 Android 端用户能看到当前数据文件实际存放地址。
- 保持本地优先：没有配置或无法访问 COS 时，应用仍可使用本地数据正常运行。
- 为同步过程定义数据上传、下载、冲突处理、失败重试和用户可见状态。
- 调整现有数据持久化需求，使本地数据保存与云同步协作，但不破坏当前 `%APPDATA%\Forge` 数据保留语义。
- 不引入破坏性变更；现有桌面端本地数据和手动导入导出能力必须继续可用。

## Capabilities / 能力范围

### New Capabilities / 新增能力

- `安卓移动端应用`: 覆盖 Android App 的安装包交付、启动入口、移动端运行环境、核心页面可用性、离线使用和与现有业务数据的集成范围。
- `腾讯云COS数据同步`: 覆盖 COS 配置、数据上传下载、同步状态、冲突处理、失败恢复、安全边界和跨端数据连续性。

### Modified Capabilities / 修改能力

- `data-persistence`: 本地数据持久化需要增加与云同步协作的需求，包括本地优先、同步失败不丢本地数据、同步写入不绕过现有数据保留和显式清理规则。

## Impact / 影响范围

- 影响应用交付形态：需要新增 Android 端构建、运行和发布相关工程配置。
- 影响数据层：需要为现有任务、OKR、能力、反思、情绪、时间块、灵感和配置等应用状态定义统一同步载荷。
- 影响存储边界：桌面端继续使用现有本地数据目录，Android 端需要对应的本地持久化位置，并与 COS 同步流程对接。
- 影响系统页展示：数据备份卡片需要在桌面端和手机端窄屏下清晰展示长本机路径，不提供编辑入口。
- 影响依赖和配置：可能引入 Android 构建工具、移动端运行时、腾讯云 COS SDK 或受控同步接口。
- 影响安全策略：用户可选择签名服务或本地手动密钥；手动密钥只保存在本机配置中，不得写入仓库、APK、云端同步快照或历史快照。
- 影响云端配置：浏览器和 Android WebView 直连 COS 时依赖 Bucket CORS，至少需要放行 GET、PUT、HEAD 和 `Content-Type` 预检。
- 影响测试：需要覆盖桌面端本地持久化回归、Android 端核心页面可用性、离线场景、同步成功、同步失败和冲突场景。
