# Data Backup System Scope Fix Design / 数据备份归属系统页修复设计

## Context / 背景

系统页曾通过 `DataBackupRitual` 提供备份导入导出能力。周看板额外渲染 `DataBackupPanel`，造成页面职责混乱和入口重复；后续确认二者属于同一功能，应收敛为系统页的单一“数据备份”入口。

## Goals / Non-Goals / 目标与非目标

目标：

- 周看板聚焦任务执行、今日进度、每日反思和可选模块。
- 数据备份只保留在系统页，并保持可被用户直接识别。
- 不改变备份业务逻辑。

非目标：

- 不调整系统页整体布局。
- 不改造备份数据格式。
- 不保留重复备份入口。

## Decisions / 设计决策

1. 从 `Dashboard` 移除数据备份组件的 import 和渲染。
2. 在 `System` 中渲染普通 `DataBackupPanel`，标题展示为“数据备份”。
3. 移除 `DataBackupRitual`，避免同一功能出现两个入口。
4. 数据备份按钮使用“导出数据 / 导入数据”直观文案。

## Risks / Trade-offs / 风险与取舍

- 用户需要进入系统页进行备份，这是更符合页面职责的路径。
- 去掉风格化入口会少一点趣味文案，但能降低功能重复和误解。
