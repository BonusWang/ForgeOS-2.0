## Context / 背景

Forge-OS 当前移动端入口已经从桌面页面压缩版，演进为 `MobileAppShell` 的四段式工作台：`今日`、`推进`、`记录`、`系统`。Android App 通过原生 Java WebView 壳加载 Vite production assets，并通过 `window.androidStorage` 写入应用私有文件；React 端和桌面端共享 Zustand store、任务数据、反思数据、灵感数据和 COS 同步载荷。

这次反馈集中在移动端日常闭环：记录时间显示与本地时区不一致、推进页不能直接完成/维护任务、不能查看历史周、记录页把灵感和反思作为同级分类。实现上应优先复用现有 store 行为，避免引入移动端专属数据模型。

## Goals / Non-Goals / 目标与非目标

**Goals / 目标:**

- 让移动端记录流按设备本地时间展示，不再直接截取 UTC ISO 字符串。
- 让移动端推进页支持任务完成、取消完成、编辑描述、移动到明天、收纳和删除。
- 让移动端推进页支持切换周，并能查看历史周任务与历史周复盘。
- 让移动端记录页形成“快速捕捉 + 结构化沉淀”的层级，降低灵感与反思同级平铺造成的误导。
- 让灵感捕捉形成“想法 -> 来源 -> 标签 -> 确认”的递进流程，鼓励补充来源和标签但不强制。
- 让移动端反思严格写入每日反思库，并复用桌面端模板字段；成效不再作为独立移动输入类型。
- 让 Android 输入法和可访问性更友好：软键盘顶起 WebView，掌控感使用下拉选择，WebView 不强制固定字体缩放。
- 让移动端模块切换进入浏览器历史，配合 Android 返回手势。
- 保持 Android 离线可用：所有操作先写本地 store，COS 同步按既有机制后续处理。

**Non-Goals / 非目标:**

- 不新增联网校时服务，不要求 Android App 主动联网更新时间。
- 不新增复杂任务编辑字段，如能力点、标签、日期选择器或优先级。
- 不重做桌面端周看板、周复盘、反思库页面。
- 不修改任务、反思、灵感的数据结构和 COS 同步 envelope。
- 不引入新的 UI 库或 Android 原生 Compose/XML 页面。

## Decisions / 设计决策

- **本地时间展示只改展示层。** 存储继续使用 ISO UTC 字符串，保证跨端同步和冲突判断稳定；移动端记录流通过 `new Date(createdAt)` 转为本地日期时间。没有选择“联网校时”，因为这会引入网络依赖、失败状态和权限/厂商后台策略成本，而当前问题是展示格式错误，不是系统时钟源缺失。
- **推进页任务操作使用行内展开。** 每个任务行保留完成作为最高频操作，编辑、明天、收纳、删除放在展开区域。没有选择复制桌面拖拽看板，因为 Android 单手场景下拖拽多列成本高，且本次目标是补齐核心闭环。
- **周切换使用轻量三按钮。** 推进页顶部提供上一周、本周、下一周；切换后同一时间线按选中周重新计算任务和周复盘。没有选择完整日历弹窗，避免首版范围过大。
- **记录页改为主次层级。** 快速捕捉区默认服务灵感；反思和成效作为“今日沉淀”区域的明确操作，仍写入同一套反思模板。没有选择删除现有三类能力，因为灵感、反思、成效都仍有价值，问题在层级而不是能力本身。
- **灵感使用递进输入。** 移动端先展示想法输入，之后再进入来源、标签和确认步骤。来源和标签保持可选，但通过节点化流程提醒用户补全上下文；没有选择一次性长表单，因为手机上长表单会降低临时捕捉意愿。
- **反思对齐每日模板。** 移动端反思直接按桌面端每日反思模板输入最大障碍、解决方法、有效/无效、明天调整和掌控感。成效作为反思内容的一部分，不再单独作为同级记录入口，避免反思被错误保存到灵感库。
- **每日反思一天一条。** 如果当天已有每日反思，移动端先展示“今日反思已保存”提示，再允许查看/编辑原反思。这样与桌面端“一天一条每日反思”的数据模型保持一致。
- **数值输入使用下拉。** 掌控感使用 1-10 select，并用小字说明 `1`、`5`、`10` 的含义。没有选择数字输入框，因为 Android 数字键盘在部分输入法下容易形成 `03` 且删除体验差。
- **日期卡片可收起。** 推进页默认展开今天或有活跃任务的日期；用户点击日期头可收起或展开该天明细，使查看周五后继续看周六时不必长距离滚动。
- **移动导航写入 history。** 底部导航切换时 push hash 状态，Android 返回手势可回到上一个移动模块；首次进入用 replace 保持入口稳定。
- **COS 手动同步不按时间戳自动覆盖。** 当本地和云端都发生变化时，即使某一端更新时间更晚，也标记冲突并保留本地备份，避免移动端和桌面端互相覆盖独立编辑。
- **删除任务需要二次确认。** 使用轻量确认态或确认按钮，避免移动端误触直接删除；删除后立即反映在本周统计中。

## Risks / Trade-offs / 风险与取舍

- Android 软键盘可能遮挡编辑输入 → 编辑区域保持在任务行内部，输入框短小，保存/取消按钮触控高度不低于 44px。
- 历史周切换后用户可能误以为能编辑历史复盘全文 → 本次只提供历史周查看和轻量复盘保存，完整复盘仍保留桌面端主路径。
- 任务删除误触风险高 → 删除操作放在次级区域，并提供确认动作。
- 移动端操作增多可能让推进页变拥挤 → 默认只展开有任务的日期和用户点击的日期，任务次操作收进详情区域。
- 设备系统时间如果本身错误，展示仍会错误 → 本次只修复 UTC 展示问题；若后续需要，可在系统页增加“设备时间异常提示”，但不在本次引入。
- 灵感递进步骤可能比一次保存多一步 → 允许来源和标签跳过，并在确认页保存，保留低摩擦捕捉。
- 一天一条反思限制可能让用户想追加多段内容时感到受限 → 本轮优先保持桌面端数据模型一致，追加内容通过编辑原反思完成。
