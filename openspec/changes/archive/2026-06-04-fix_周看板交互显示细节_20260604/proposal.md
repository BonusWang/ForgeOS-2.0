# Dashboard Interaction Detail Fix / 周看板交互显示细节修复

## Why

> 中文：为什么

周看板中的两个视觉交互细节影响了日常使用的可读性：用户点击日历日期时，弹出框会在页面滚动后严重偏离日期格；今日进度条使用字符绘制，切换字体或风格后空进度段会出现异常视觉块，并且在今日进度卡片里显得过短。

## What Changes

> 中文：改动内容

- 日历日期弹出框使用 fixed 视口坐标时，不再叠加页面滚动偏移。
- 日历弹出框在靠近视口边缘时保持在可见区域内。
- 今日进度条从字符拼接改为 CSS 轨道和填充层渲染，避免 `░` 字符造成宽度和纹理异常。
- 今日进度卡片中的进度条占据可用宽度的约 80%，减少大面积空白。
- 保留进度百分比文案和原有任务完成统计逻辑。

## Impact

> 中文：影响范围

- 影响周看板中的日历弹出框定位和今日进度条视觉呈现。
- 不改变任务、日历事项、习惯、反思、模块启用或持久化数据结构。
- 通用进度条仍可被能力面板复用，今日进度卡片使用专用样式扩展宽度。

## Out of Scope

> 中文：暂不包含

- 不重做日历交互模型。
- 不调整任务完成统计规则。
- 不改变周看板模块顺序或业务数据来源。
