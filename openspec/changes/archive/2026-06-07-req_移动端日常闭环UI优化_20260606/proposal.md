## Why / 背景

当前 Android/手机端已经具备今日、推进、记录和系统入口，但若干交互仍像“桌面能力搬到手机”：同步状态展示同时暴露旧基线和 V3 基线，推进页可以收纳却无法在手机端处理收纳箱，记录页的灵感/反思反馈混在同一个记录流里，展开输入后也缺少收起路径。

本次变更要把移动端从“能操作”补齐到“能独立完成日常闭环”，同时明确所有 UI 优化不得回退现有保存、同步、任务维护和离线能力。

## What Changes / 变更内容

- 系统页 COS 同步状态主区域只展示当前有效的 V3 同步信息，旧基线不再与 V3 基线同级展示。
- 「推进」页在周日下方新增跨周公共收纳箱，展示所有未安排任务。
- 收纳箱支持新增、编辑、删除和安排到日期；安排时通过底部面板优先选择当前查看周 7 天，也可选择其他日期。
- 「推进」页任务操作按钮保留编辑、明天、收纳、删除能力，但调整为更轻、更适合手机列表扫读的 UI。
- 「记录」页灵感捕捉改为纵向递进节点：`01 想法 -> 02 来源 -> 03 标签 -> 04 确认`。
- 「记录」页写灵感和写反思支持展开后收起，不再依赖切换模块恢复收起状态。
- 「记录」页取消混合记录流；灵感卡片展示最近灵感，反思卡片展示最近反思。
- 系统页风格设置新增移动端本机独立风格开关；默认跟随桌面，开启后移动端风格不影响桌面端风格。
- 明确本次不是数据模型大改造：任务、灵感、反思继续复用现有业务数据流，COS 日常同步继续使用 V3 实体文档。

## Capabilities / 能力范围

### New Capabilities / 新增能力

- 无。

### Modified Capabilities / 修改能力

- `手机端网页适配`: 补充移动端推进收纳箱、任务操作 UI、记录页纵向流程、独立最近列表、展开收起和移动端风格独立设置要求。
- `安卓移动端应用`: 补充 Android 端任务收纳公共池、记录页闭环和本机风格偏好不得破坏离线与返回导航的要求。
- `腾讯云COS数据同步`: 调整同步状态展示要求，并补充移动端本机风格偏好不应误入跨端同步的要求。

## Impact / 影响范围

- 主要影响 `src/features/mobile/MobileWeekProgress.tsx`、`src/features/mobile/MobileCaptureHub.tsx`、`src/features/mobile/MobileAppShell.tsx`、`src/features/system/SyncPanel.tsx`、`src/App.tsx` 和 `src/index.css`。
- 可能影响移动端本机风格偏好的存储位置；如需新增字段，必须与跨端同步的全局 config 区分。
- 需要更新 `tests/mobileProductStructure.test.ts`、`tests/syncPanelStructure.test.ts` 等结构测试。
- 不应改动 COS V3 合并引擎、Android 原生 WebView 启动方式、桌面端任务看板或桌面端反思库的数据语义。
