# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

## [2.1.0] - 2026-06-17

### Added
- 新增轻量浏览器端 launcher：Go 编写的单文件 exe（约 9MB），双击在系统默认浏览器运行 Forge-OS，与 Electron 桌面端共享同一份本地数据，前端零改动。
- 新增 `npm run launcher:build` 一键构建 launcher exe（`vite build` → 复制 `dist` → `go build` 内嵌）。
- 新增浏览器端一键启动脚本 `start-browser.bat` / `scripts/dev-launch.ps1`（先清理 Vite 端口上的旧 dev server 再启动并自动打开浏览器，只清理 5173-5180 端口，不误伤其他 node 应用）。
- 新增 `npm run dev:open` 启动开发服务器并自动打开浏览器。

### Changed
- README 补充 launcher 作为轻量分发选项的说明，以及与 Electron 桌面端的体积/形态取舍（~100MB → ~9MB）。
- 发布版本升级到 2.1.0。

## [2.0.1] - 2026-06-10

### Added
- 新增 Multica 自动化研发协作流水线文档，沉淀 Forge-OS issue、agent、评审、测试和发布阶段的协作规则。

### Changed
- V3 同步在仍有字段冲突时也会把无冲突的远端实体合并进本地状态，避免安全变更被冲突阶段挡住。

### Fixed
- 修复浏览器和 Android 同时完成同一任务时仅 `completedAt` 不同却生成冲突的问题；现在会保留较新的完成时间并自动合并。
- 修复已经不再复现的旧 V3 字段冲突会继续残留在同步 envelope 中的问题。

## [2.0.0] - 2026-06-07

### Added
- Forge OS 2.0 作为独立大版本首发，当前发布版本从 `v2.0` 开始计算。
- 新增 `docs/releases/v2.0.md`，集中说明 2.0 的产品定位、发布范围和验证记录。

### Changed
- 工程版本升级到 `2.0.0`，Android 版本同步升级到 `versionName 2.0.0`、`versionCode 20`。
- README 补充 Forge OS 2.0 独立大版本定位和当前发布版本说明。

### Fixed
- 保留桌面端周复盘保存后的“周复盘已保存”摘要反馈，并作为 2.0 首发体验的一部分发布。

## [1.0.8] - 2026-06-07

### Changed
- README 补充周复盘保存后的可见反馈说明，明确保存成功后会进入已保存摘要，并可继续查看/编辑。

### Fixed
- 修复桌面端周复盘点击“保存周复盘”后没有任何可见反馈的问题；保存成功后现在展示“周复盘已保存”摘要。
- 已保存周复盘提供“查看/编辑周复盘”入口，避免用户保存后不知道是否还能继续调整内容。

## [1.0.7] - 2026-06-07

### Added
- 新增 Claude 视觉系统说明和 OpenSpec 变更，用于约束视觉风格只改变表现层，不改变页面结构、业务模块或数据语义。
- 新增手机端「记录」页的「灵感 / 反思」分段信息架构，两类记录共享纵向流程链但保持字段、文案、保存路径和最近记录列表独立。
- 新增产品设计专家 skill，并沉淀 Forge 与 matrix-life-os 的专家团评审报告、桌面端样例图、移动端样例图和成长闭环样例图。

### Changed
- Claude 视觉风格替换旧暖色参考风格，桌面端和手机端共享更新后的暖奶油画布、珊瑚主色、细边框和字体栈。
- 旧 `orbit` 风格偏好读取时自动映射为 `claude`，避免已有用户的风格选择失效。
- 手机端「今日」改为轻量「今日焦点」状态卡，今日任务新增入口从右下角大型 FAB 收敛到「今日承诺」标题右侧。
- 手机端今日承诺任务行进一步紧凑化，减少顶部状态卡和任务区之间的首屏挤压。

### Fixed
- Android 构建同步 Vite Web 资源前会清理旧生成目录，避免历史资源残留进 APK。
- 手机端反思草稿收起保护改为基于反思答案变化判断，减少无变更反思面板的误提示。

## [1.0.6] - 2026-06-07

### Added
- 新增 V3 实体级 COS 同步引擎，日常跨端同步优先使用 `forge-data.entities.v3.json`。
- 新增 V3 同步命名空间和 profile 隔离，默认对象路径进入 `profiles/<profileId>/`。
- 新增三方合并、删除 tombstone、字段级冲突和冲突去重能力，减少 Android 与浏览器并发修改时的手工选择。
- 系统同步面板新增 V3 namespace、revision、初始化状态、自动合并数量、冲突数量和最近错误展示。
- 新增 Dossier 视觉风格，并支持手机端使用独立于桌面端的视觉风格偏好。

### Changed
- 手动同步、启动同步和自动同步切换为 V3 runner 优先；v1 快照和历史快照保留为手工备份、恢复或回退通道。
- 手机端「今日」改为轻量「今日焦点」加「今日承诺」行动台，新增任务入口收敛到今日承诺标题右侧。
- 手机端「记录」改为「灵感 / 反思」分段切换，两者复用同一交互框架但保持文案、字段、保存路径和最近记录独立。
- 灵感和反思流程均使用纵向流程链展示当前、已完成和未开始状态。
- 数据备份从历史快照恢复后会清理 V3 基线状态，并提示需要重新初始化或刷新 V3 基线。
- OpenSpec 正式规格同步 V3 同步引擎、移动端记录页和今日页体验要求。

### Fixed
- 修复旧 v1/v2 主同步对象存在时可能被误当作 V3 初始化来源的问题；现在会拒绝自动迁移并提示清理或手工恢复。
- 修复远端 V3 envelope namespace 或 profileId 不匹配时可能串用数据的问题；现在会拒绝导入。
- 修复无本地 V3 基线的移动端首次同步可能把远端基线当成本地待上传变更的问题。
- 修复历史恢复后继续沿用旧 V3 base 可能导致合并判断不准确的问题。

## [1.0.5] - 2026-06-05

### Added
- 手机端「今日」新增右下角快速添加任务入口，可在当前页面直接创建今天的 active 任务。
- 手机端「推进」支持上一周、本周、下一周切换，并展示历史周任务和已保存周复盘摘要。
- 手机端「记录」的灵感收集改为想法、来源、标签、确认保存的递进工作流，来源和标签可跳过。
- 新增安卓移动端设计需求专家 skill，用于后续 Android 原生需求分析、交互评审和兼容性决策。

### Changed
- 手机端底部导航写入浏览器历史，Android 返回手势可回到上一个移动模块。
- 手机端「推进」任务行支持完成、取消完成、编辑描述、移动到明天、收纳和确认删除。
- 手机端日期卡片支持展开和收起，查看后续日期时不必被已展开任务列表阻挡。
- 手机端反思输入对齐桌面每日反思模板，反思保存到反思库，成效归入反思内容字段。
- 掌控感改为 1-10 下拉选择，并补充数值说明，避免 Android 数字键盘产生 `03` 等异常输入。
- Android WebView 保留系统字体缩放，并通过 `adjustResize` 避让软键盘。
- OpenSpec 正式规格同步移动端核心闭环、今日快速添加任务、灵感递进流、反思模板输入和 COS 冲突策略。

### Fixed
- 修复移动端记录时间直接截取 UTC ISO 导致本地显示偏差的问题。
- 修复移动端反思可能被保存进灵感库、再次打开默认展示旧反思内容且缺少提示的问题。
- 修复 COS 手动同步在本地和云端都变化时可能按更新时间自动覆盖另一端的问题；现在会进入冲突状态并保留本地备份。
- 修复系统同步面板将待上传本地变更误读成“已是最新”的文案问题。

## [1.0.3] - 2026-06-04

### Added
- 新增 Android App 工程，支持打包内置 Vite 生产资源的 Forge-OS 移动端应用。
- 新增 `npm run android:build`、`npm run android:install` 和 `npm run android:smoke`，用于构建、安装和采集 Android 冒烟测试截图。
- 新增移动端专用工作台，提供“今日、推进、记录、系统”底部导航，并复用桌面端同一套本地数据。
- 新增腾讯云 COS 数据同步能力，支持签名服务和本机手动密钥两种授权方式。
- 新增 COS 主同步对象、历史备份快照、冲突推荐、首次同步选择、启动同步和本地变更自动同步。
- 新增 `npm run cos:signer` 签名服务脚本和 `.env.example` 配置模板。

### Changed
- 系统页新增 COS 数据同步面板，并在数据备份区域展示当前平台只读本机存储地址。
- 桌面端左侧导航加入“月度 OKR”，系统入口和风格/模块/主题工具收敛到底部区域。
- 模块管理、周看板、周复盘、反思库和系统页补充移动端样式，减少窄屏重叠和横向溢出。
- OpenSpec 正式规格同步 Android 应用、Android 冒烟测试、腾讯云 COS 同步和本地优先持久化要求。

### Fixed
- 云同步快照排除手动 Access Key 和 Secret Access Key，避免密钥进入云端载荷。
- COS 同步失败时保留本地数据和导入导出入口，继续维持本地优先使用体验。
- 补齐归档 change `2026-06-02-fix_风格切换不改变页面形态_20260602` 的 `design.md` 和 `tasks.md`。
