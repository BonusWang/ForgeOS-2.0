# Forge-OS v2.1.0 Release Notes

发布日期：2026-06-17

## 本次重点

v2.1.0 新增轻量浏览器端 launcher。在此之前，Forge-OS 的可执行分发只有 Electron 桌面端，单文件 exe 约 100MB——其中 90% 以上是 Electron 自带的 Chromium + Node runtime，业务代码本身只有 2.7MB。launcher 用 Go 编写一个约 9MB 的单文件 exe，双击后在系统默认浏览器里运行 Forge-OS，不再打包整个 runtime，体积下降约 91%。由于复用同一份本地数据文件，launcher 与 Electron 桌面端数据互通，切换形态不丢数据。

## 更新内容

- 新增 launcher：Go 编写的单文件 exe（约 9MB），双击启动本地 HTTP 服务并打开系统默认浏览器。
- launcher 内嵌前端静态资源（`embed`），实现 `/__forge_data__` 存储接口（契约与 vite dev middleware 一致），数据原子写入并带 `.bak` 备份。
- launcher 与 Electron 桌面端共享 `%APPDATA%/Forge/forge-data.json`，前端零改动（`platformStorage` 的 localhost 分支自动生效）。
- 新增 `npm run launcher:build` 一键构建：`vite build` → 复制 `dist` → `go build`（embed + `-ldflags "-s -w"`）。
- 新增浏览器端一键启动脚本 `start-browser.bat` / `scripts/dev-launch.ps1`：先清理 Vite 端口上的旧 dev server（只清理 5173-5180，不误伤其他 node 应用），再启动并自动打开浏览器。
- 新增 `npm run dev:open` 启动开发服务器并自动打开浏览器。
- README 补充 launcher 作为轻量分发选项的说明，及其与 Electron 桌面端的体积与形态取舍。

## 体积对比

| 分发方式 | 产物体积 | 运行形态 |
|---|---|---|
| Electron 桌面端 (`electron:build`) | ~100MB | 原生桌面窗口 |
| 浏览器端 launcher (`launcher:build`) | ~9MB | 系统浏览器标签页 |

## 验证

- `npm run build`
- `npm run launcher:build` → 产出 `release/Forge-OS.exe`（8.8MB）
- launcher 端到端：启动 → 自动开浏览器 → 静态首页返回 200 → 存储 GET/PUT/DELETE 往返一致 → 数据落盘并生成 `.bak`

## 兼容性

- 不修改任务、反思、OKR、灵感或同步配置的任何数据字段。
- launcher 与 Electron 桌面端、vite dev server 共享同一数据文件路径与格式，数据互通。
- V3 同步、Android 私有存储、localStorage 迁移层均未改动。
- 本次未包含 Android 变更。
