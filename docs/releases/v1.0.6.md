# Forge-OS v1.0.6 Release Notes

发布日期：2026-06-07

## 本次重点

v1.0.6 将 Forge-OS 的 COS 日常同步升级为 V3 实体级同步：浏览器和 Android 默认读写同一份带命名空间的 V3 实体文档，并通过三方合并、删除 tombstone 和字段级冲突记录减少跨端并发修改时的手工恢复。同时，手机端「今日」和「记录」页进一步收敛为更轻量的行动路径，系统页补充 V3 同步状态，现有 Dossier 视觉风格和手机端独立风格偏好也一并进入发布版本。

## 更新内容

- 新增 V3 主同步对象 `forge-data.entities.v3.json`，日常同步优先使用实体级同步文档。
- V3 对象路径按 `profiles/<profileId>/` 隔离，并在 envelope 中校验 namespace 和 profileId，避免不同配置串用数据。
- 同步引擎单独维护 V3 本地基线、revision、checksum、删除 tombstone 和冲突列表，不向业务对象写入同步字段。
- 新增 `base + local + remote` 三方合并，可自动合并不同实体、不同字段和同值并发修改。
- 新增字段级冲突、删除/编辑冲突和冲突去重，重复同步不会反复生成同一冲突。
- 旧 v1/v2 主同步对象不再作为 V3 自动迁移来源；如果旧对象仍存在，系统会提示清理或手工恢复。
- 手动同步、启动同步和自动同步切换为 V3 runner 优先，v1 快照和历史快照保留为手工备份、恢复或回退通道。
- 系统同步面板展示 V3 namespace、revision、初始化状态、自动合并数量、冲突数量和最近错误。
- 从本地导入或历史快照恢复后会清理 V3 基线状态，并提示重新初始化或刷新 V3 基线。
- 手机端「今日」改为轻量「今日焦点」状态卡和独立「今日承诺」任务区。
- 手机端新增任务入口从右下角大型悬浮按钮调整到「今日承诺」标题右侧的小型 `+` 入口。
- 手机端「记录」页改为「灵感 / 反思」分段切换，用户无需向下滚动即可进入反思。
- 灵感与反思复用标题、收起/展开、纵向流程链、输入区和最近记录区，但字段、文案、保存路径和记录列表保持独立。
- 灵感流程保留想法、来源、标签、确认；反思流程保留障碍、方法、有效/无效、确认。
- 新增 Dossier 视觉风格，并支持手机端选择独立于桌面端的视觉风格偏好。
- OpenSpec 归档 V3 同步引擎 change，并补充移动端记录页与今日页体验要求。

## 兼容性

- 本次更新不修改任务、反思、心情、灵感、目标等业务实体字段。
- V3 同步不自动读取或迁移旧云端 v1/v2 主同步对象；旧快照仍可作为手工恢复材料。
- 未配置 COS、网络不可用或 V3 同步失败时，本地优先读写继续可用。
- Android 与浏览器默认共享同一个 profile 的 V3 同步对象；不同 profileId 会写入不同对象路径。
