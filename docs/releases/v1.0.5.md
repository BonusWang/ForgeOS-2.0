# Forge-OS v1.0.5 Release Notes

发布日期：2026-06-05

## 本次重点

v1.0.5 聚焦 Android/手机端从“能看”走向“能独立完成日常闭环”：今日页可以直接新增任务，推进页可以完成、维护和跨周查看任务，记录页把灵感收集改为更适合手机的递进流程，反思输入则对齐桌面每日反思模板。同时，COS 手动同步在本地和云端同时变化时不再自动覆盖任一端。

## 更新内容

- Android/手机端「今日」新增右下角 `+` 快速添加任务入口，输入非空内容后直接创建今天的 active 任务。
- 「推进」页支持上一周、本周、下一周切换，可查看历史周任务和已保存周复盘摘要。
- 「推进」页任务支持完成、取消完成、编辑描述、移动到明天、收纳和确认删除。
- 「推进」页日期卡片可点击展开和收起，查看周五后继续看周六不再需要长距离滚动。
- 「记录」页灵感收集改为想法、来源、标签、确认保存的递进工作流，来源和标签可跳过。
- 灵感记录保存后会保留来源和自定义标签，并在最近保存记录流中展示。
- 移动端反思输入按桌面每日反思模板填写，反思保存到反思库，不再误入灵感库。
- 成效归入反思内容字段，不再作为移动端独立同级记录类型。
- 掌控感改为 1-10 下拉选择，并补充 `1`、`5`、`10` 的含义说明。
- 手机端底部导航写入浏览器历史，Android 返回手势可回到上一个移动模块。
- Android WebView 保留系统字体缩放，并通过 `adjustResize` 避让软键盘。
- 系统同步面板将本地待上传状态展示为“待上传本地变更”，不再误写成“已是最新”。
- COS 手动同步在本地和云端都变化时进入冲突状态并保留本地备份，不再只按更新时间自动上传或恢复。
- 新增 Android 移动端设计需求专家 skill，沉淀后续移动端需求评审角色。
- OpenSpec 正式规格同步并归档移动端核心闭环增强与今日快速添加任务两个 change。

## 验证

- `openspec validate --all --strict --no-interactive`
- `node --test tests\mobileProductStructure.test.ts tests\androidProject.test.ts tests\manualSync.test.ts`
- `npm run lint`
- `npm run build`
- `npm run android:build`

## 兼容性

- 本次更新不修改任务、灵感、反思或 COS 快照的数据结构。
- Android 本地私有存储路径保持不变，离线读写继续可用。
- COS 同步载荷保持 v1 snapshot 结构；冲突处理策略更保守，以避免跨端独立编辑被自动覆盖。
- 桌面端工作台入口和已有桌面布局不因本轮移动端增强而回退。
