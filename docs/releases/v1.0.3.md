# Forge-OS v1.0.3 Release Notes

发布日期：2026-06-04

## 本次重点

v1.0.3 把 Forge-OS 从桌面端工作台推进到移动端和云同步：新增 Android App 工程、手机端专用工作台、腾讯云 COS 数据同步，以及 Android 冒烟测试脚本。核心原则仍然是本地优先，云同步只负责备份、恢复和跨端衔接。

## 更新内容

- 新增 Android App 工程，可打包内置前端生产资源，不依赖 Vite dev server。
- 新增移动端底部导航，包含“今日、推进、记录、系统”四个入口。
- 新增 COS 数据同步面板，支持签名服务和本机手动密钥两种授权方式。
- 新增首次同步选择、手动同步、冲突推荐、历史快照备份和启动同步。
- 新增 `npm run android:build`、`npm run android:install`、`npm run android:smoke` 和 `npm run cos:signer`。
- 系统页数据备份区域展示当前平台只读本机存储地址。
- 桌面端导航加入“月度 OKR”，系统入口和工具入口收敛到底部。
- OpenSpec 正式规格同步 Android、COS 同步、Android 冒烟测试和本地优先持久化要求。
- 补齐风格切换归档 change 缺失的 `design.md` 和 `tasks.md`。

## 验证

- `openspec validate --all`
- OpenSpec 归档文档完整性审计
- `npm run lint`
- `npm run build`
- `node --test tests/*.test.ts`：77 tests, 77 pass

## 兼容性

- 桌面端数据目录继续使用 `%APPDATA%\Forge\forge-data.json`。
- Android 端使用应用私有本地存储，离线时仍可读写本地数据。
- COS 同步配置中的手动密钥仅保存在本机，不写入云端同步快照。
- COS 未配置或同步失败时，本地导入、导出和保存能力不受影响。
