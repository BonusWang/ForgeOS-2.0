# Forge-OS v1.0.8 Release Notes

发布日期：2026-06-07

## 本次重点

v1.0.8 聚焦桌面端周复盘的保存反馈。此前用户点击“保存周复盘”后，数据会写入反思记录，但页面没有任何明确提示，容易让用户误判为没有保存或按钮没有响应。本次将保存后的表单收起为已保存摘要，并保留“查看/编辑周复盘”入口，让保存动作有明确回声，也让后续修改路径更清楚。

## 更新内容

- 桌面端周复盘保存成功后展示“周复盘已保存”摘要。
- 已保存摘要展示本周完成和下周调整的轻量预览。
- 已保存摘要提供“查看/编辑周复盘”入口，用户可以重新打开表单继续修改。
- 新增 `aria-live="polite"` 和 `role="status"`，让保存状态可被辅助技术感知。
- Android 版本号同步更新到 `1.0.8`，versionCode 更新到 `8`。
- README 补充周复盘保存反馈说明。

## 验证

- `node --test tests\weeklyReviewSaveFeedback.test.ts`
- `node --test <all tests/*.test.ts>`
- `npm run build`
- 浏览器验证：打开桌面端周复盘，填写必填项并点击保存；确认出现“周复盘已保存”摘要，并可通过“查看/编辑周复盘”重新打开表单。

## 兼容性

- 本次更新不修改周复盘、反思、任务或同步数据结构。
- 保存逻辑仍复用现有 `saveReflection`，只增加桌面端交互反馈。
- Android 和移动端周复盘原有保存反馈保持不变。
