# Forge-OS v2.0.1 Release Notes

发布日期：2026-06-10

## 本次重点

v2.0.1 聚焦 COS V3 同步冲突体验和 Forge-OS 的 agent 研发协作流程。同步层现在会在冲突阶段继续吸收可安全合并的远端实体，减少浏览器和 Android 并发使用时的“明明无关却没同步过来”的情况；任务完成时间也会按较新的时间自动合并，不再制造无意义冲突。

## 更新内容

- V3 同步在仍有字段冲突时，也会把无冲突的远端任务、反思等实体应用到当前本地状态。
- 浏览器和 Android 同时把同一任务标记为完成、但 `completedAt` 略有差异时，会保留较新的完成时间并自动合并。
- 当前合并已经不再复现的旧字段冲突会从 V3 envelope 中清理，避免冲突计数长期残留。
- 新增 Multica 自动化研发协作流水线文档，覆盖项目经理、需求专家、开发、集成测试、UI/交互评审、交互测试、交付和发布阶段。
- Android 版本号同步更新到 `versionName 2.0.1`，`versionCode 21`。
- README 和 CHANGELOG 同步更新到 v2.0.1。

## 验证

- `node --test tests\v3MergeEngine.test.ts tests\v3SyncRunner.test.ts`
- `node --test tests\*.test.ts`
- `npm run lint`
- `npm run build`

## 兼容性

- 本次更新不修改任务、反思、OKR、灵感或同步配置的数据字段。
- V3 同步对象路径、namespace、profile 隔离和 checksum 校验保持不变。
- 未配置 COS 或网络不可用时，本地优先读写继续可用。
